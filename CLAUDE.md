# Cosplay Taobao - Project Guide

## Overview

A full-stack web app for saving and browsing Taobao cosplay listings. Users paste JSON data from a Taobao product listing, the app extracts relevant fields and saves them to MySQL, then displays all saved items in a card-based grid with images.

## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | Bun |
| Backend | Elysia (TypeScript web framework) |
| Frontend | React 19 + TypeScript |
| UI | Mantine v8 |
| Forms | Mantine Form (`@mantine/form`) |
| State | Jotai |
| Data Fetching | TanStack React Query + Eden Treaty (type-safe API client) |
| Routing | TanStack Router (file-based) |
| Database | MySQL via `bun:sql` |
| Linter/Formatter | Biome |
| Virtualization | TanStack React Virtual |
| Image processing | sharp |

## Database Schema

Single table: **`items`**

| Column | Type | Notes |
|---|---|---|
| `id` | INT AUTO_INCREMENT PK | |
| `image_url` | VARCHAR(2083) NULL | Multiple URLs separated by `\|\|` |
| `original_title` | VARCHAR(2083) NOT NULL | Raw title from Taobao JSON |
| `custom_title` | VARCHAR(2083) NULL | User-edited display title |
| `seller_name` | VARCHAR(255) NULL | |
| `listing_url` | VARCHAR(2083) NOT NULL | Taobao product URL |
| `notes` | TEXT NULL | Free-form user notes |
| `original_price` | VARCHAR(255) NULL | Price string from listing |
| `is_archived` | TINYINT(1) DEFAULT 0 | Soft-delete flag |
| `archived_at` | TIMESTAMP NULL | |
| `created_at` | TIMESTAMP DEFAULT CURRENT_TIMESTAMP | |
| `updated_at` | TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | |

Schema is created automatically on startup via `initDb()` in [src/backend/db.ts](src/backend/db.ts).

## API

- Base: `/api`
- OpenAPI docs: `/api/docs`
- `GET    /api/items`            ‚Äî list all items
- `POST   /api/items`            ‚Äî create item; body `{ json: string, override?: boolean }`; returns 409 on duplicate URL
- `PATCH  /api/items/:id`        ‚Äî update `custom_title` and/or `notes`
- `DELETE /api/items/:id`        ‚Äî hard delete (also removes local thumb files)
- `POST   /api/items/:id/archive`   ‚Äî soft-archive
- `POST   /api/items/:id/unarchive` ‚Äî unarchive
- Backend exports `App` type; frontend imports it for Eden Treaty inference

## Key Conventions

- **Path alias**: `@/*` maps to `./src/*`
- **API client**: Always use the Eden Treaty client in [src/frontend/api.ts](src/frontend/api.ts) ‚Äî it's fully type-safe against the Elysia backend
- **Adding routes**: Add files under `src/frontend/routes/` and run `generate-routes`; never edit `routeTree.gen.ts`
- **Formatting**: Biome with 2-space indent, double quotes, no semicolons
- **Backend pattern**: Controller (`index.ts`) ‚Üí Service (`service.ts`) ‚Üí Model types (`model.ts`) inside each feature folder
- **Image thumbnails**: `image_url` may contain multiple `||`-delimited URLs; `public/thumbs/` holds locally cached copies (UUID filenames). Local paths start with `thumbs/` ‚Äî `resolveImages()` in `model.ts` prepends `/` to make them root-relative. `downloadThumbs.ts` skips URLs that already contain `thumbs` (already local).
- **Archiving**: Items use soft-delete (`is_archived` + `archived_at`); no hard deletes via the UI (DELETE endpoint exists for scripting)
- **Duplicate detection**: Backend strips URL after first `&` and checks `listing_url LIKE %baseUrl%`; frontend shows a confirmation modal on 409
- **Static file serving**: `decodeURIComponent` is applied to the pathname before `Bun.file()` so filenames with spaces (stored as `%20` in URLs) resolve correctly

## Frontend ‚Äî Index Page (`routes/index.tsx`)

- **Sticky toolbar**: JSON paste input + pink "Add Item" button (Enter key supported); Divider; debounced search input (`useDebouncedValue`, 300ms); "Show Archived Only" Chip
- **Item grid**: `VirtualCardGrid` with `ItemCard` components
- **ItemCard**: Mantine Carousel for images, archived gray border + badge + date, title, seller, price, button row pinned to bottom of card
  - "View Listing" ‚Äî opens Taobao URL in new tab
  - "Notes" ‚Äî only shown when notes exist; opens read-only modal
  - "Edit" ‚Äî opens form modal to edit `custom_title` and `notes`
- **Modals** (all defined once in `IndexPage`, shared across cards):
  - Duplicate confirmation modal (409 response)
  - Notes view modal (read-only, scrollable)
  - Edit modal (`useForm` with `custom_title` + `notes` fields, calls `PATCH /api/items/:id`)
